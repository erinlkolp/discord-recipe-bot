# Bugfix Plan: upsert_guild placement, ingredient casing, category safety

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix three bugs: uncommitted `upsert_guild` on early-return paths, lowercased ingredient names in shopping list, and potential KeyError in `shopping_view` for unknown categories.

**Architecture:** All three are small, isolated fixes. Bug 1 requires moving `upsert_guild` calls after validation guards (or removing them where unnecessary). Bug 2 requires a `display_names` dict in `aggregate_shopping_items`. Bug 3 requires using `setdefault` instead of direct key access.

**Tech Stack:** Python, SQLAlchemy, discord.py, pytest

```python
def aggregate_shopping_items(items: list[dict]) -> list[dict]:
    """Aggregate and scale ingredient quantities for a shopping list.

    Each item dict must have keys:
        name, quantity (Decimal|None), unit, category,
        entry_servings (int), recipe_servings (int)

    Returns list of dicts with keys: ingredient_name, total_quantity, unit, category.
    Items with the same (name.lower(), unit) are summed after scaling.
    Different units for the same ingredient produce separate line items.
    """
    totals: dict[tuple, Decimal] = {}
    categories: dict[tuple, str] = {}
    display_names: dict[tuple, str] = {}
    ordered_keys: list[tuple] = []
    seen: set[tuple] = set()

    for item in items:
        scale = Decimal(str(item['entry_servings'])) / Decimal(str(item['recipe_servings']))
        key = (item['name'].lower(), item['unit'] or '')
        if key not in seen:
            seen.add(key)
            ordered_keys.append(key)
            display_names[key] = item['name']
        if item['quantity'] is not None:
            totals[key] = totals.get(key, Decimal(0)) + item['quantity'] * scale
        categories[key] = item['category']

    return [
        {
            'ingredient_name': display_names[key],
            'unit': key[1],
            'total_quantity': totals.get(key),
            'category': categories[key],
        }
        for key in ordered_keys
    ]
```
